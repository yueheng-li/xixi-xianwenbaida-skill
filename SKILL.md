# 溪溪仙问百答 — 儿童科学内容量产编排器

> **IP**：《天庭幼儿园：今天神仙老师教什么》—— 7岁科学课代表闹闹 + 翻车神仙。
> **定位**：儿童科学百问百答。闹闹淡定认真 vs 神仙咋咋呼呼 = 核心笑点。
> **形态**：🎬 抖音短视频（30-50秒）| 🎧 有声书（10-15分钟）。

## 路由表

| 用户意图 | 调度 |
|----------|------|
| 做一期完整视频/音频 | → 按 [核心链路](#核心链路) 逐步执行 |
| 只写文案 | → `Skill("xixi-文案创作")` |
| 已有文案，做分镜+配图 | → `Skill("xixi-分镜与配图")` |
| 已有分镜，做配音+合成 | → `Skill("xixi-配音与合成")` |
| 已有成片，优化/归档/发布 | → `Skill("xixi-发布归档")` |
| 批量生产 N 条 | → [量产模式](#量产模式) |
| 从中间续做 | → [断点续做](#断点续做) |
| 首次使用（定妆照） | → `Skill("xixi-分镜与配图")` 的 Step 0A |

## 内容形态速查

| 形态 | 时长 | 字数 | 开篇 |
|------|------|------|------|
| 🎬 视频 | 30-50秒 | 150-300字 | 翻车/问题/神仙提问开场 |
| 🎧 音频 | 10-15分钟 | 1800-2500字 | **必须**"哈喽小伙伴们！欢迎收听月恒的有声书。今天聊——xxx" |

## 合规铁律

> ⚠️ 所有 prompt 生成前必须通过 [`_shared/合规校验标准.md`](_shared/合规校验标准.md) 逐项检查。

## 核心链路

```
① 文案创作 ──→ ② 分镜+配图 ──→ ③ 配音+合成 ──→ ④ 发布归档
   (xixi-文案创作)  (xixi-分镜与配图)  (xixi-配音与合成)  (xixi-发布归档)
```

**执行规则**：
1. 每一步调用对应的子技能（`Skill("xixi-xxx")`）
2. 当前步骤的输出作为下一步骤的输入
3. **文案是唯一人工闸门**——必须用户审核通过后才能继续
4. 每步完成后告知用户当前进度，询问是否继续

## 触发条件

触发词：`仙问百答` / `闹闹` / `天庭幼儿园` / `神仙教科学` / `月恒` / `有声书` / `儿童科普视频`

**类型识别**（首先执行）：
- 用户说"视频"/"抖音" → 🎬 视频模式
- 用户说"音频"/"有声书"/"月恒" → 🎧 音频模式
- 没明确指定 → **必须问**："你要视频版还是音频版？"

## 快速启动·视频模式

```
用户：帮我做一期仙问百答视频，知识点是"为什么放屁会臭"

执行：
  Skill("xixi-文案创作") → 生成30-50秒对话体文案 → 用户审核通过
  → Skill("xixi-分镜与配图") → 分镜JSON + 动画分镜 + 配图 + 动画prompt
  → Skill("xixi-配音与合成") → TTS多角色配音 → HyperFrames合成 → 成品MP4
  → Skill("xixi-发布归档") → 优化报告 + Obsidian归档 + 发布包
```

## 快速启动·音频模式

```
用户：帮我做一期仙问百答音频，知识点是"为什么放屁会臭"

执行：
  Skill("xixi-文案创作") → 生成10-15分钟有声书文案 → 用户审核通过
  → Skill("xixi-配音与合成") → TTS多角色配音+拼接 → 成品MP3
  → Skill("xixi-发布归档") → 优化报告 + Obsidian归档 + 发布包
```

## 量产模式

日更 3-5 条时：
1. 知识点批量提取 → 2. `Skill("xixi-文案创作")` 批量生成 → 3. 流水线并行推进分镜/配音/配图
4. 渲染队列逐个 → 5. `Skill("xixi-发布归档")` 批量归档

**效率参考**：单条约 15-25 分钟，日均量产 3-5 条。

## 质量检查清单

渲染前：文案字数 ✓ / 科学准确 ✓ / 至少1笑点 ✓ / 分镜speaker完整 ✓ / 配图风格统一 ✓ / 合规通过 ✓
